# アプリ詳細ドキュメント

このドキュメントは Hackit 2026 の申請フォームアプリ（フロントエンド: Next.js / バックエンド: Convex）について、開発者向けに設計・構成・重要ファイル・実行手順・運用上の注意点をまとめたものです。

## 目的
- イベント参加申請やチーム情報の登録を受け付ける Web アプリケーション。
- シンプルな UI から Convex を利用してデータを安全に保存・検証する。

## 主な機能
- 申請フォーム（イベント／チーム情報）送信
- 送信内容のサーバー側検証（API ルート）
- 期限バナー表示やフォーム制御などの UI コンポーネント

## 技術スタック
- Next.js (App Router)
- React, TypeScript
- Convex（バックエンドのデータ層 / 関数）
- TailwindCSS（スタイル）

## アーキテクチャ概要
- UI 層: `app/` フォルダ内のページとコンポーネントがレンダリングを担当。
- API 層: `app/api/.../route.ts` に Next.js API ルートが実装され、必要に応じて Convex と連携。
- Convex: `convex/` フォルダにスキーマ、関数、生成ファイルが格納される。

簡易フロー:
1. ユーザーがフォーム送信
2. Next.js API またはクライアント側で初期検証
3. Convex のミューテーションを呼び出して保存
4. 成功/失敗をフロントへ返却

## 重要ファイル（概要）
- `app/page.tsx` — トップページ（フォーム一覧や説明）
- `app/components/EventForm.tsx` — 申請フォーム UI
- `app/components/DeadlineBanner.tsx` — 締切バナーコンポーネント
- `app/api/submission/verify/route.ts` — 申請内容の検証 API
- `app/api/teams/verify/route.ts` — チーム情報検証 API
- `convex/schema.ts` — Convex のスキーマ定義
- `convex/events.ts` — Convex 側の関数やクエリ（イベント処理）
- `convex/_generated/` — Convex の自動生成ファイル（編集禁止）

（ファイル位置はルートからの相対パスです。該当ファイルを直接参照してください）

## 環境変数・設定
- ローカルで Convex を使う場合、`npx convex dev` による開発サーバー起動で大半はローカル接続できます。
- デプロイ環境では Convex のデプロイ URL や API キー、必要に応じた認証情報を環境変数に設定してください。

※ 具体的な環境変数名はプロジェクトのデプロイ設定（ホスティングやConvexの設定）によります。プロダクションにデプロイする前に `convex/README.md` の手順を確認してください。

## ローカル開発手順（復習）
1. 依存関係をインストール

```bash
npm install
```

2. Convex 開発サーバー（別ターミナル）

```bash
npx convex dev
```

3. Next.js 開発サーバー起動

```bash
npm run dev
```

4. ブラウザで `http://localhost:3000` にアクセス

## デプロイの注意点
- Convex を利用する場合、Convex 側のスキーマや関数を本番環境にデプロイしておく必要があります（`npx convex deploy` 等）。
- Next.js のビルド時に環境変数が正しく設定されていることを確認してください。
- 自動生成ファイル（`convex/_generated/`）はビルド成果物に含めるかどうかは運用方針次第ですが、通常はリポジトリに含めておくとローカル開発が容易になります。

## テスト・静的解析
- ESLint を導入済みです。変更後に `npm run lint` を実行してください。
- 単体テストのフレームワークはこのリポジトリに含まれていないため、必要なら `vitest` や `jest` を追加してください。

## 貢献方法
- 変更を行う際は最初に issue を作成し、簡単な変更方針を共有してください。
- Pull Request は小さく分け、説明に変更点と意図を明記してください。

## 運用上のヒント
- Convex のクエリ・ミューテーションは出来るだけ単純に保ち、バリデーションはサーバー側（Convex / API ルート）でも再実施すること。
- 大量のデータ読み書きが予想される場合は、Convex のドキュメントや料金・レート制限に留意してください。
